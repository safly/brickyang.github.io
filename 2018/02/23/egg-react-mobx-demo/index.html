如何使用 React、Webpack、MobX、TypeScript 和 Egg.js 技术栈

Feb 23, 2018 | Hits

本文通过一个 Todo List 的 DEMO 介绍如何使用 React、Webpack、MobX、TypeScript 和 Egg.js 技术栈进行 SSR（服务端渲染）App 的开发。DEMO 代码位于 egg-react-mobx-demo，包含了三个逐步「演化」的版本：

  • v1：非 TypeScript，与 Egg.js 非耦合。可以将 Server 换成任何一种框架（Express、Koa 等）。
  • v2：非 TypeScript，与 Egg.js 耦合。
  • current：前后端使用 TypeScript 重写。

这套技术栈中，React 和 Webpack 是关键。MobX 只是一个状态管理工具，可以替换为 Redux 或者不用。这里加入 MobX 纯粹只是想试试与 Redux 不同的工具。

Egg.js 也不是严格必须的，可以替换为其他服务端框架，比如 Express 或 Koa。但是从第二阶段开始会把 webpack-dev-server 整合进 Egg.js。

v1

v1 版本是最「正统」的架构，在开发环境下使用独立的 webpack-dev-server
